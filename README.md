# ut-vet ЁЯФН

**Unit Test Validator** тАФ Catch meaningless tests before they waste your CI.

`ut-vet` is a static analysis tool that detects unit tests that compile and pass but don't actually validate anything. It uses AST parsing to find common anti-patterns in test code, especially those generated by AI tools. Supports **Go** and **Rust**.

## Installation

```bash
go install github.com/NaughtyIshmael/ut-vet/cmd/ut-vet@latest
```

## Quick Start

```bash
# Analyze Go test files recursively
ut-vet ./...

# Analyze Rust test files
ut-vet src/

# Analyze specific paths (Go + Rust auto-detected)
ut-vet ./pkg/handlers/ ./src/models/

# Include all severity levels
ut-vet --severity p2 ./...

# JSON output for CI
ut-vet --format json ./...
```

## What It Detects

### P0 тАФ Critical

| Rule | Description | Example |
|------|-------------|---------|
| `empty-test` | Test body is empty or comments-only | `func TestFoo(t *testing.T) {}` / `fn test_foo() {}` |
| `no-assertion` | Test has no assertion calls | Test does computation but never checks results |
| `log-only-test` | Test only logs/prints, never asserts | `t.Log("ok")` / `println!("ok")` |
| `trivial-assertion` | Assertion on constant/literal values | `assert.True(t, true)` / `assert!(true)` |

### P1 тАФ High Value

| Rule | Description |
|------|-------------|
| `error-not-checked` | Returned error assigned to `_` or unchecked |
| `no-code-under-test` | Test never calls functions from the tested package |
| `zero-value-input` | Function called with only zero-values (`nil`, `0`, `""`) |
| `only-nil-check` | Only checks `err == nil`, ignores actual result |

### P2 тАФ Advanced

| Rule | Description |
|------|-------------|
| `tautological-assert` | Assert variable equals itself (`assert_eq!(x, x)`) |
| `dead-assertion` | Assertion after `t.Fatal`/`return`/`panic!` (unreachable) |
| `no-arrange` | No meaningful setup тАФ all args are nil/zero |
| `happy-path-only` | Only tests success path of a fallible function |

## Mutation Testing

Static analysis catches structurally weak tests, but can't tell if tests actually catch code logic errors. Use `ut-vet mutate` to run mutation testing тАФ it injects small code changes and checks if your tests detect them.

```bash
# Run mutation testing (auto-detects Go/Rust)
ut-vet mutate .

# With minimum score threshold (fail CI if below 80%)
ut-vet mutate --threshold 0.8 .

# JSON output
ut-vet mutate --json .

# Verbose (show all mutants, including killed)
ut-vet mutate -v .
```

**Prerequisites:**
- **Go**: Install [gremlins](https://github.com/go-gremlins/gremlins): `go install github.com/go-gremlins/gremlins/cmd/gremlins@latest`
- **Rust**: Install [cargo-mutants](https://github.com/sourcefrog/cargo-mutants): `cargo install --locked cargo-mutants`

## Usage

```
ut-vet [flags] [paths...]

Flags:
  --format text|json     Output format (default: text)
  --rules <ids>          Comma-separated rule IDs to enable (default: all)
  --exclude <patterns>   Comma-separated glob patterns to exclude files
  --severity p0|p1|p2    Minimum severity to report (default: p0)
  --list-rules           List all available rules
  --version              Show version
  -v                     Verbose output
  -q                     Quiet mode (findings only)

Subcommands:
  mutate [flags] [path]  Run mutation testing via gremlins (Go) or cargo-mutants (Rust)
```

## CI Integration

`ut-vet` exits with code 1 when issues are found, making it easy to use as a CI gate:

```yaml
# GitHub Actions
- name: Vet unit tests
  run: |
    go install github.com/NaughtyIshmael/ut-vet/cmd/ut-vet@latest
    ut-vet ./...
```

## Language Support

- тЬЕ **Go** тАФ Full AST-based analysis via `go/ast` (parses `_test.go` files)
- тЬЕ **Rust** тАФ Text-based parser (detects `#[test]` and `#[tokio::test]` functions in `.rs` files)

Both languages are auto-detected тАФ just point ut-vet at your source tree.
