# ut-vet 🔍

**Unit Test Validator** — Catch meaningless tests before they waste your CI.

`ut-vet` is a static analysis tool that detects unit tests that compile and pass but don't actually validate anything. It uses AST parsing to find common anti-patterns in test code, especially those generated by AI tools. Supports **Go** and **Rust**.

## Installation

```bash
go install github.com/NaughtyIshmael/ut-vet/cmd/ut-vet@latest
```

## Quick Start

```bash
# Analyze Go test files recursively
ut-vet ./...

# Analyze Rust test files
ut-vet src/

# Analyze specific paths (Go + Rust auto-detected)
ut-vet ./pkg/handlers/ ./src/models/

# Include all severity levels
ut-vet --severity p2 ./...

# JSON output for CI
ut-vet --format json ./...
```

## What It Detects

### P0 — Critical

| Rule | Description | Example |
|------|-------------|---------|
| `empty-test` | Test body is empty or comments-only | `func TestFoo(t *testing.T) {}` / `fn test_foo() {}` |
| `no-assertion` | Test has no assertion calls | Test does computation but never checks results |
| `log-only-test` | Test only logs/prints, never asserts | `t.Log("ok")` / `println!("ok")` |
| `trivial-assertion` | Assertion on constant/literal values | `assert.True(t, true)` / `assert!(true)` |

### P1 — High Value

| Rule | Description |
|------|-------------|
| `error-not-checked` | Returned error assigned to `_` or unchecked |
| `no-code-under-test` | Test never calls functions from the tested package |
| `zero-value-input` | Function called with only zero-values (`nil`, `0`, `""`) |
| `only-nil-check` | Only checks `err == nil`, ignores actual result |

### P2 — Advanced

| Rule | Description |
|------|-------------|
| `tautological-assert` | Assert variable equals itself (`assert_eq!(x, x)`) |
| `dead-assertion` | Assertion after `t.Fatal`/`return`/`panic!` (unreachable) |
| `no-arrange` | No meaningful setup — all args are nil/zero |
| `happy-path-only` | Only tests success path of a fallible function |

## Mutation Testing

Static analysis catches structurally weak tests, but can't tell if tests actually catch code logic errors. Use `ut-vet mutate` to run mutation testing — it injects small code changes and checks if your tests detect them.

```bash
# Run mutation testing (auto-detects Go/Rust)
ut-vet mutate .

# With minimum score threshold (fail CI if below 80%)
ut-vet mutate --threshold 0.8 .

# JSON output
ut-vet mutate --json .

# Verbose (show all mutants, including killed)
ut-vet mutate -v .
```

**Prerequisites:**
- **Go**: Install [gremlins](https://github.com/go-gremlins/gremlins): `go install github.com/go-gremlins/gremlins/cmd/gremlins@latest`
- **Rust**: Install [cargo-mutants](https://github.com/sourcefrog/cargo-mutants): `cargo install --locked cargo-mutants`

## Usage

```
ut-vet [flags] [paths...]

Flags:
  --format text|json     Output format (default: text)
  --rules <ids>          Comma-separated rule IDs to enable (default: all)
  --exclude <patterns>   Comma-separated glob patterns to exclude files
  --severity p0|p1|p2    Minimum severity to report (default: p0)
  --list-rules           List all available rules
  --version              Show version
  -v                     Verbose output
  -q                     Quiet mode (findings only)

Subcommands:
  mutate [flags] [path]  Run mutation testing via gremlins (Go) or cargo-mutants (Rust)
```

## GitHub Action

Use ut-vet as a reusable GitHub Action in any project — no manual install needed:

```yaml
- uses: NaughtyIshmael/ut-vet@main
  with:
    paths: './...'
    severity: 'p0'
```

Findings appear as inline PR annotations automatically.

### Action Inputs

| Input | Default | Description |
|-------|---------|-------------|
| `paths` | `.` | Space-separated paths to analyze |
| `mode` | `analyze` | `analyze` (static analysis) or `mutate` (mutation testing) |
| `severity` | `p0` | Minimum severity: `p0`, `p1`, or `p2` |
| `format` | `text` | Output format: `text` or `json` |
| `rules` | (all) | Comma-separated rule IDs to enable |
| `exclude` | (none) | Comma-separated glob patterns to exclude files |
| `verbose` | `false` | Enable verbose output |
| `quiet` | `false` | Quiet mode — findings only |
| `version` | `latest` | ut-vet version (e.g. `v1.0.0` or `latest`) |
| `mutation-tool` | (auto) | `gremlins` (Go) or `cargo-mutants` (Rust) |
| `mutation-threshold` | (none) | Minimum mutation score (0.0–1.0) |

### Action Outputs

| Output | Description |
|--------|-------------|
| `exit-code` | `0` = clean, `1` = findings detected, `2` = error |

### Examples

```yaml
# Basic — P0 checks on all test files
- uses: NaughtyIshmael/ut-vet@main
  with:
    paths: './...'

# Strict — all severity levels, specific rules
- uses: NaughtyIshmael/ut-vet@main
  with:
    severity: 'p2'
    rules: 'empty-test,no-assertion,trivial-assertion'
    paths: './pkg/...'

# JSON output, excluding generated files
- uses: NaughtyIshmael/ut-vet@main
  with:
    format: 'json'
    exclude: '*_generated_test.go,*_mock_test.go'
    paths: './...'

# Mutation testing with a score threshold
- uses: NaughtyIshmael/ut-vet@main
  with:
    mode: 'mutate'
    mutation-threshold: '0.8'
    paths: '.'
```

> **Note:** The action requires Go to be set up in your workflow (e.g. via `actions/setup-go`).

## CI Integration (Manual)

If you prefer to install ut-vet directly instead of using the Action:

```yaml
- name: Vet unit tests
  run: |
    go install github.com/NaughtyIshmael/ut-vet/cmd/ut-vet@latest
    ut-vet ./...
```

`ut-vet` exits with code 1 when issues are found, making it easy to use as a CI gate.

## Language Support

- ✅ **Go** — Full AST-based analysis via `go/ast` (parses `_test.go` files)
- ✅ **Rust** — Text-based parser (detects `#[test]` and `#[tokio::test]` functions in `.rs` files)

Both languages are auto-detected — just point ut-vet at your source tree.
