# ut-vet 🔍

**Unit Test Validator** — Catch meaningless tests before they waste your CI.

`ut-vet` is a static analysis tool that detects unit tests that compile and pass but don't actually validate anything. It uses AST parsing to find common anti-patterns in test code, especially those generated by AI tools.

## Installation

```bash
go install github.com/NaughtyIshmael/ut-vet/cmd/ut-vet@latest
```

## Quick Start

```bash
# Analyze current directory recursively
ut-vet ./...

# Analyze specific paths
ut-vet ./pkg/handlers/ ./internal/

# JSON output for CI
ut-vet --format json ./...

# Verbose mode
ut-vet -v ./...
```

## What It Detects

### P0 — Critical

| Rule | Description | Example |
|------|-------------|---------|
| `empty-test` | Test body is empty or comments-only | `func TestFoo(t *testing.T) {}` |
| `no-assertion` | Test has no assertion calls | Test does computation but never checks results |
| `log-only-test` | Test only logs/prints, never asserts | `func TestBar(t *testing.T) { t.Log("ok") }` |
| `trivial-assertion` | Assertion on constant/literal values | `assert.True(t, true)`, `assert.Equal(t, 1, 1)` |

### Planned (P1/P2)

- `error-not-checked` — Returned error assigned to `_`
- `no-code-under-test` — Test never calls functions from the tested package
- `zero-value-input` — Function called with only zero-values
- `tautological-assert` — Assert variable equals itself
- `only-nil-check` — Only checks `err == nil`, ignores actual result

## Usage

```
ut-vet [flags] [paths...]

Flags:
  --format text|json     Output format (default: text)
  --rules <ids>          Comma-separated rule IDs to enable (default: all)
  --exclude <patterns>   Comma-separated glob patterns to exclude files
  --severity p0|p1|p2    Minimum severity to report (default: p0)
  --list-rules           List all available rules
  --version              Show version
  -v                     Verbose output
  -q                     Quiet mode (findings only)
```

## Example Output

```
src/handler_test.go:42: [no-assertion] TestCreateUser has no assertion calls
src/handler_test.go:58: [trivial-assertion] TestDeleteUser asserts a constant expression: assert.True(t, true)
src/repo_test.go:15: [empty-test] TestGetAll is empty
src/service_test.go:30: [log-only-test] TestProcess only logs/prints but has no assertions
```

## CI Integration

`ut-vet` exits with code 1 when issues are found, making it easy to use as a CI gate:

```yaml
# GitHub Actions
- name: Vet unit tests
  run: |
    go install github.com/NaughtyIshmael/ut-vet/cmd/ut-vet@latest
    ut-vet ./...
```
